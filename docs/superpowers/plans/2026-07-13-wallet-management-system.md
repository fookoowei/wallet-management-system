# Wallet Management System Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a portfolio-grade fintech wallet system with an immutable ledger, approval workflows, RBAC, and audit logging — backend (NestJS + Prisma + PostgreSQL) and frontend (Angular).

**Architecture:** A NestJS REST API backed by PostgreSQL via Prisma. Money movement is modeled as an immutable ledger of `Transaction` rows settled atomically inside database transactions. Access is gated by permission-based RBAC guards. An Angular + Angular Material web app provides the user and admin experiences.

**Tech Stack:** NestJS, TypeScript, Prisma, PostgreSQL, JWT (access + refresh), Angular, Angular Material, RxJS, Docker.

## Global Constraints

- **Node.js:** version 20 LTS or newer.
- **Package manager:** npm (lockfile committed).
- **ORM:** Prisma (single `schema.prisma` source of truth; all schema changes go through migrations — never edit the DB directly).
- **Money:** never mutate a balance directly; balances change only via settled `Transaction` rows. Settled transactions are immutable (corrections are new rows). Every balance-changing operation runs inside a Prisma `$transaction`.
- **Auth:** passwords stored as bcrypt/argon2 hashes, never plaintext. Access tokens short-lived; refresh tokens long-lived.
- **Access control:** every money/admin endpoint is gated by an explicit permission via an RBAC guard. Requesting and approving are distinct permissions (separation of duties).
- **Testing:** Jest for backend (unit + e2e). Each task ends green.
- **Commits:** frequent, one per task step where indicated; conventional-commit style messages (`feat:`, `test:`, `chore:`).
- **Money amounts:** stored as integer minor units (e.g. cents) to avoid floating-point errors — never as JS floats.

- [ ] **Step 2: Write the seed script**

Create `backend/prisma/seed.ts`:

```typescript
import { PrismaClient } from '@prisma/client';
import * as bcrypt from 'bcrypt';

const prisma = new PrismaClient();

const PERMISSIONS = [
  'deposit.approve',
  'withdrawal.approve',
  'wallet.adjust',
  'user.manage',
  'audit.view',
  'transaction.view_all',
];

const ROLE_PERMISSIONS: Record<string, string[]> = {
  super_admin: PERMISSIONS, // all
  admin: ['user.manage', 'transaction.view_all', 'audit.view'],
  finance: ['deposit.approve', 'withdrawal.approve', 'wallet.adjust', 'transaction.view_all'],
  support: ['transaction.view_all'],
  user: [],
};

async function main() {
  // 1. permissions
  for (const code of PERMISSIONS) {
    await prisma.permission.upsert({
      where: { code },
      update: {},
      create: { code },
    });
  }

  // 2. roles with their permissions
  for (const [name, codes] of Object.entries(ROLE_PERMISSIONS)) {
    await prisma.role.upsert({
      where: { name },
      update: {
        permissions: { set: codes.map((code) => ({ code })) },
      },
      create: {
        name,
        permissions: { connect: codes.map((code) => ({ code })) },
      },
    });
  }

  // 3. super-admin user
  const superAdminRole = await prisma.role.findUniqueOrThrow({ where: { name: 'super_admin' } });
  const passwordHash = await bcrypt.hash('ChangeMe123!', 10);
  await prisma.user.upsert({
    where: { email: 'admin@wallet.local' },
    update: {},
    create: {
      email: 'admin@wallet.local',
      passwordHash,
      firstName: 'Super',
      lastName: 'Admin',
      roleId: superAdminRole.id,
    },
  });

  console.log('Seed complete: permissions, roles, super-admin user.');
}

main()
  .then(() => prisma.$disconnect())
  .catch(async (e) => {
    console.error(e);
    await prisma.$disconnect();
    process.exit(1);
  });
```

import { Injectable } from '@nestjs/common';
import { PrismaService } from '../prisma/prisma.service';

@Injectable()
export class HealthService {
  constructor(private readonly prisma: PrismaService) {}

  async check(): Promise<{ status: 'ok'; db: 'up' | 'down' }> {
    try {
      await this.prisma.$queryRaw`SELECT 1`;
      return { status: 'ok', db: 'up' };
    } catch {
      return { status: 'ok', db: 'down' };
    }
  }
}
